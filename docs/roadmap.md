# Roadmap

## Phase 1: Foundation

Goal: Make CarbonForge UI credible as a package and documentation site.

- Monorepo setup.
- UI package.
- Storybook docs app.
- Next.js example app.
- CI pipeline.
- Initial `EmptyState` component.
- Accessibility and design principles docs.

## Phase 2: Enterprise Data Patterns

- Data table wrapper with toolbar conventions.
- Saved filter bar.
- Bulk action toolbar.
- Result summary strip.
- Error and permission states.
- Table skeleton and loading patterns.

## Phase 3: Workflow Patterns

- Review queue layout.
- Multi-step form wizard shell.
- Approval action panel.
- Task detail side panel.
- Audit and comment composition.
- SLA/status indicators.

## Phase 4: AI Product Patterns

- AI review panel improvements.
- Confidence indicator variants.
- Source citation list.
- AI disclosure banner.
- Prompt/result comparison layout.
- Human approval workflow pattern.

## Phase 5: Release Readiness

- npm publishing workflow.
- Changesets.
- Visual regression testing.
- Full documentation site.
- Contribution guide refinement.
- Public examples.

## Initial Milestone Definition

The first public release should include:

- A small, polished set of reusable patterns.
- Storybook coverage for each component.
- Unit or interaction test coverage for core behavior.
- Accessibility documentation.
- Next.js example app.
- CI passing.
- Clear README and usage examples.
